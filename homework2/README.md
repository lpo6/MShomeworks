## 分模块编写的思路
1.分模块编写的好处是，对于ds的错误，能及时调整，为debug提供便利。

2.先编写出主体部分，看看主页的设计、画布摆放、UI设计有无不妥，及时调整。

3.浏览主体代码，确定需要填充的部分。

4.按模块指导ds编写剩余的代码
